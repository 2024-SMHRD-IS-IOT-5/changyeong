🧡[에러케이스]

1. Objects are not vaild as a React Child (found : object with keys {}).
   - props를 작성하는 곳에 구조분해할당 {}를 쓰지 않았을 경우 -> 매개변수

2. ~~~ is not defined.
   - 'use ~' : react hook을 import하지 않았을 경우
   - '변수이름' : 변수이름을 선언할 때 오타가 난 경우
   - 아예 선언하지 않았을 경우

3. Assignment to constant variable.
   - const 키워드로 선언한 변수나 state나 함수에 새로운 값을 대입하려고 할 때 나는 오류
   - state를 const에 담는데, state에 직접적으로 값을 넣으려고 할 때 오류가 난다.
   O : setNum(3)
   X : num = 3

4. 따로 에러는 나지 않는데 화면이 아예 안뜬다?
   - 괄호 실수일 확률이 매우 높다.
   - return문을 반환해야하는데, 위에서 괄호 실수가 나서 return문을 먹어버린 경우 多
   - 왼쪽에 있는 화살표로 작은 단위부터 하나씩 접어나가기 시작하면 어디가 문제인지 보임

5. JSX elements must be wrapped ~
   - return문 안에는 하나의 큰 부모요소가 감싸져야함

6. export 'default' was not found in '경로'
   - 경로 안에서 그 컴포넌트를 찾을 수 없는 경우
   - 파일이 없는 경우
   - 파일은 있는데 export를 안한경우
   - 실수로 파일을 소문자로 만들었다가 대문자로 이름을 바꾸는 과정에서
     어떤 특정 지점만 여전히 소문자라서 그 경로를 못찾는 경우

7. React Hook "use~" cannot be called at the top level.
   React Hooks must be called in a React function components
   or a custom React Hook Function.
   - 리액트 훅은 컴포넌트 내 생성자 위치에서만 선언되어야 한다.
   - 괄호 실수로 인하여 component의 분리가 일어나버린 경우

8. ~.map is not a function, .filter is not a function ~
   - map, filter "배열"함수임 배열을 대상으로만 쓸 수 있음
   - map의 대상이 배열이 아닌 경우 나는 오류
   - 배열 state의 초기값을 배열이 아닌 undefined로 뒀을 때
   - 배열을 세팅하는 과정에서 오타가 나는 경우
   - 배열의 이름에 오타가 난 경우
   - 복잡한 구조의 데이터를 사용할 때, 배열이 아닌 다른 데이터타입을 썼을 경우

9. 오류는 안나는데, 이상하게 두번 눌러야만 실행이 되거나 
   내가 원하는 결과가 한박자 늦게 나오는 경우
   - useEffect를 통해 실행되는 순서를 다시 지정해주어야한다.

10. Module not Found : Error : Can't resolve '~~~' in '~~'
   - 설치 안한 것
   ex) react-router-dom, react-bootstrap, axios .. => npm i 이름

   - 재설치를 했는데도 오류가 난다? 오타일 확률
   ex) Can't ~~ "react-router-don"
   
   - 설치 했는지 안했는지 정확하게 확인하고 싶다면?
     해당 프로젝트의 package.json으로 가면 설치여부, 버전 확인 가능

11. Type Error : ~ is not iterable.
   - Object is not iterable?  
     props를 쓸 때 {속성1, 속성2} 이렇게 적어줘야하는데 [속성1, 속성2] 이렇게 적은 경우
   
   - 변수 is not iterable?
   ex) const [list, setList] = useEffect() (X)
       let [list, setList] = [1,2,3] (X)
       <Context.Provider value={값1, 값2}> (X)


💛[함수 사용법]

- 리액트는 함수 호출이 아닌 함수 참조형식으로 함수를 실행한다.
- 함수 호출 : 함수이름() -> 즉시 실행이 되어버림
- 함수 참조 : 함수이름 -> 이벤트 발생 시 리액트에게 함수 실행을 맡기는 거임

- 그러려면 매개변수를 어떻게 전달할 것인가? 콜백함수를 통해 전달
- ()=>{함수호출진행}

- ex) 함수이름이 Func
- 매개변수 없이 함수 실행 : onClick={Func}
- 매개변수가 필요하다? : onClick{()=>{Func(매개변수)}}

💖cors오류 (node.js)

   - 포트번호가 다르면 컴퓨터 입장에서는 다른사람이라고 인지
   - "너 누구야? 일단 나가" == CORS오류

   1. npm i cors (노드프로젝트에서)
   2. index.js 파일로 이동해서 app.use(cors())
   2-2. app.use(cors({
            origin : "내가 허가할 주소",
            credentials : true
         }))
   3. index.js 파일에 express.json()
